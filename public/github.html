Pull Requests

Use this form to select and download Github pull requests. Use the "Repo Group", "Repo Status", and "Service Type" to help further filter based on the repo group information.  If not using repo group, clear the repo group field and just use the PR Filter/Query string.  


Toggle Help

Pull Request Form Guide

  • Repository Group - Select a repo group to run the PR filter against.  Clear - Clears the selection box if you do not want to use repo groups.
  • Repo Status - Select the repo status to further filter the repo groups.
    • Active - RM managed repository
    • Inactive - Repository is no longer used
    • Code Artifact - Developer/DevOps managed repository. (Not RM managed)
    • All - All statuses
  • Service Type - Select the location of the platform
    • Enterprise - Only Github repos within the group that are hosted on Enterprise Server
    • Cloud - Only Github repos within the group that are hosted on Cloud Server
  • PR Query Filter - Use the Github filter syntax to identify pull requests
  • # of PR's - Shows the number of pull requests that meets all the criteria selected.
  • Get Data - Submits query to Github API to retrieve Pull Request Information
  • Show PR Table - Creates a table view of the select PR's. 
  • NonComplieant - Creates a table view of PR's that do not have proper format of Jira# in PR title, Missing Milestone on PR, etc.

Repository Groups

Toggle Help

Repo Groups give the app the ability to pull info from GitHub based on groupings of repositories. The app uses the repo group to  identify which repos to pull data.  There are several metadata items in the group that can act as a filter. Multiple groups can be included in the JSON fill. Metadata includes the following:

  • Group: An identifying name for the group 
  • Owner: The Github repo owner/organization.
  • RepoName: The name of the Repository
  • RepoType: Indicates if the repo is GitHub "Cloud" or GitHub "Enterprise"
  • Status: (A)ctive - Repos owned by RM, (I)nactive - Repos no longer used, (C)odeArtifact - Developer owned Repos
  • Division: Company Division, Department, etc.   [PCG, CBG, TAG], 
  • Platform: Used to identify type of system platform related to the repo. [AEM, SW, AWS, OMS,...]
